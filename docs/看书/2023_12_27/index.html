  • 面向对象编程: 创建一个基本的 JavaScript 对象 | freeCodeCamp.org
  • 面向对象编程的内容看过了一遍，只在这个网页里面看过了一遍，整体上的理解非常浅薄。后面还需要做的事情是看另外一个网页的教程，以及看其他的JS书籍。也还需要回过头来看这里面的内容。
  • 不仅可以从父辈那里继承行为，还可以通过mixin的方式获得共通的行为，比如birds和airplane都有飞行的行为。
  • mixin 是一个包含可被其他类使用而无需继承的方法的类。
  • mixin可以把一个类的方法拷贝到其他类，而无需有继承关系。
  • 在JS中mixin代表什么意思？
    • 在 JavaScript 中，mixin 是一种编程模式，用于将一个对象的属性和方法混合（或合并）到另一个对象中，以便在目标对象中拥有来自不同源的特性。
    • 通过 mixin，可以将一个或多个对象的特性合并到另一个对象，从而创建具有多种功能的新对象。这种模式可以帮助避免对象继承的复杂性，并支持代码的重用。
    • 在 JavaScript 中，可以使用不同的方法来实现 mixin，例如使用 Object.assign()、扩展运算符（...）、Object.setPrototypeOf() 等。通常情况下，mixin 可以用于实现代码的模块化、组合特性以及提供可复用的功能。
    • 下面是一个简单的 JavaScript 示例，展示了如何使用 mixin：
    • // 定义一个 mixin 对象
      const myMixin = {
      sayHello() {
        console.log('Hello!');
      },
      sayGoodbye() {
        console.log('Goodbye!');
      }
      };
      - // 目标对象
      const myObject = {};
      - // 将 mixin 对象的特性混合到目标对象中
      Object.assign(myObject, myMixin);
      - // 使用混合后的对象
      myObject.sayHello(); // 输出: Hello!
      myObject.sayGoodbye(); // 输出: Goodbye!
    • 在上面的例子中，myMixin 对象包含了 sayHello 和 sayGoodbye 方法，然后通过 Object.assign() 将这些方法混合到了 myObject 对象中，使得 myObject 能够调用这些方法。
    • 需要注意的是，在使用 mixin 时，可能会出现属性或方法名冲突的情况。因此，合理命名、组织和管理 mixin 对象非常重要，以避免潜在的命名冲突和意外覆盖现有特性。
    • 在使用 mixin 的过程中，对象的名称不需要包含 "mixin" 这样的关键字。你可以根据实际情况为对象选择任何合适的名称，没有特定的命名要求。通常，对象名称应该清晰、具有描述性，能够反映该对象的作用或所承载的功能。
    • Mixin 可以被赋予任何名称，而且这个名称不一定要包含 "mixin"。实际上，为了代码的可读性和可维护性，你应该选择一个有意义的名称来描述这个 mixin 提供的功能或特性。
    • 举个例子，假设你有一个用于处理日志的 mixin：
    • const logMixin = {
      logMessage(message) {
        console.log(`Log: ${message}`);
      },
      logError(error) {
        console.error(`Error: ${error}`);
      }
      };
      - // 可以将 logMixin 合并到任何对象中，比如一个名为 logger 的对象
      const logger = {};
      - // 将 logMixin 的功能混合到 logger 对象中
      Object.assign(logger, logMixin);
      - // 现在 logger 对象可以使用 logMixin 中的方法
      logger.logMessage('This is a log message.'); // 输出: Log: This is a log message.
      logger.logError('An error occurred.'); // 输出: Error: An error occurred.
    • 在这个例子中，我们将一个名为 logMixin 的 mixin 合并到了一个名为 logger 的对象中，这个对象可以使用 mixin 中定义的日志方法。虽然 mixin 的名称是 logMixin，但被混合的对象名可以是任意的，只要它符合你的代码结构和逻辑。
  • 在面向对象中，this相当于一个代词，用来指代与这个方法相关的这个对象的内容。
  • 闭包：
    • 变量作用域：在JS中，在中括号{...}里面申明的变量，只在这个中括号{...}里面可见，在这个中括号外面不可见。在if，while，for循环的中括号也是一样的。
    • 在JS中，闭包是嵌套的内部函数，他可以访问外部作用域，外部函数的变量，但外部无法直接访问闭包里面的变量，从而保护闭包内部的变量不会轻易被修改。外部函数只能通过调用闭包，来间接访问闭包内的变量。
