    • [基础数据结构: 使用 for 循环遍历数组中的全部元素 | freeCodeCamp.org]
    • 错误的代码
    • function filteredArray(arr, elem) {
    • let newArr = [];
    • // 只修改这一行下面的代码
    • for (let i=0;i<arr.length;i++) {
    • for (let j=0;j<arr[i].length;j++){
    • if(arr[i][j]==elem){
    • break;}
    • }
    • newArr.push(arr[i])
    • }
    • // 只修改这一行上面的代码
    • return newArr;
    • }
    • console.log(filteredArray([[7, 2, 3], [1, 6, 3], [3, 13, 26], [19, 3, 9]], 7));
    • 上面的代码是错的

      • 在提供的代码中，newArr.push(arr[i]) 语句位于外部 for 循环内部的 if 语句之后，这导致无论内部的 if 条件是否为真，都会将 arr[i] 添加到 newArr 中。这意味着在发现 elem 元素后，依然会将包含该元素的子数组添加到 newArr 中，而不会过滤掉包含 elem 元素的子数组。 background-color:: yellow

      • 在这种情况下，需要确保只有当不包含 elem 元素时才将子数组添加到 newArr 中。修正方法是在发现 elem 元素后，跳过将其添加到 newArr 的步骤。

      • 下面是更正后的代码：

      • function filteredArray(arr, elem) {
        let newArr = [];

        for (let i = 0; i < arr.length; i++) {
          let containsElem = false; // 标记是否包含 elem 元素
        - for (let j = 0; j < arr[i].length; j++) {
            if (arr[i][j] === elem) {
              containsElem = true; // 如果找到 elem 元素，则标记为包含
              break; // 停止内部循环，因为已经找到了 elem 元素
            }
          }
        - if (!containsElem) {
            newArr.push(arr[i]); // 只有当不包含 elem 元素时才添加到新数组中
          }
        }

        return newArr;
        }
        - console.log(filteredArray([[7, 2, 3], [1, 6, 3], [3, 13, 26], [19, 3, 9]], 7));

      • 这段修正后的代码会检查是否包含 elem 元素，并且只有在不包含 elem 元素的情况下才会将子数组添加到 newArr 中。

  • 在一个嵌套了不知道多少层的数组中，提供遍历所有元素的方法
    • 在 JavaScript 中，你可以使用递归来遍历嵌套的多层数组中的所有元素。递归是一种在函数内部调用自身的技术。
    • 以下是遍历多层嵌套数组的一个示例：
    • function iterateNestedArray(arr) {
      for (let i = 0; i < arr.length; i++) {
        if (Array.isArray(arr[i])) {
          iterateNestedArray(arr[i]); // 递归调用：如果当前元素是数组，则继续遍历其内部元素
        } else {
          console.log(arr[i]); // 处理非数组元素（假设这里只是简单打印）
        }
      }
      }
      - const nestedArray = [1, [2, [3, 4]], [5, 6], [[[7]]]];
      iterateNestedArray(nestedArray);
    • 上述示例中的 iterateNestedArray 函数会递归地遍历嵌套的多层数组，如果当前元素是数组，则会再次调用 iterateNestedArray 来处理该子数组，直到遍历完所有嵌套的元素。
    • Array.isArray() 是 JavaScript 中的一个方法，用于确定给定的值是否是数组。当 Array.isArray() 方法的参数是一个数组时，它将返回 true；否则，返回 false。
    • 在这个特定的代码段中，arr[i] 表示取数组 arr 的第 i 个元素，然后 Array.isArray(arr[i]) 用来检查这个元素是否是数组类型。如果 arr[i] 是数组类型，条件表达式 Array.isArray(arr[i]) 将会返回 true，否则返回 false。
    • 请注意，这只是一个简单的示例，假设数组中的元素是简单类型的。在实际情况中，可能需要根据元素的类型执行不同的操作，比如针对对象、函数等的处理方式会有所不同。
    • 递归是一种强大的技术，但需要注意避免潜在的无限递归问题和性能问题，特别是在处理大型数据结构时。
  • 下面一个错误我当初怎么都没有搞清楚，没有搞清楚变量和属性名的差异。变量只能用方括号【】，属性名可以用方括号或者点记法

    • 1

    • 这两段代码都是用于计算具有 online 属性且值为 true 的用户数量。两段代码的实现逻辑基本相同，但存在一些差异：

    • 来源：[基础数据结构: 使用 for...in 语句遍历对象 | freeCodeCamp.org]

    • 第一段代码：​

      function countOnline(allUsers) {
      let i = 0;
      for (let user in allUsers) {
        if (allUsers.user.online === true) {
          i++;
        }
      }
      return i;
      }

      这段代码中的问题在于 for...in 循环中，使用了 allUsers.user.online。for...in 循环中的 user 是一个变量，表示 allUsers 中的属性名，而不是属性本身。因此，allUsers.user 实际上是在尝试获取 allUsers 对象中名为 'user' 的属性值，而不是遍历到的当前用户对象。所以条件判断 allUsers.user.online === true 无法正确访问用户对象的 online 属性。

    • 第二段代码：​

      function countOnline(usersObj) {
      let result = 0;
      for (let user in usersObj) {
        if (usersObj[user].online === true) {
          result++;
        }
      }
      return result;
      }

      这段代码更为正确。它使用 for...in 循环遍历 usersObj 对象的属性，user 是属性名，在每次循环中获取了对应属性值 usersObj[user]，然后判断该对象的 online 属性是否为 true。如果满足条件，则将 result 计数器加一。

    • 总结差异：​

    • 第一段代码中的错误在于使用了 allUsers.user.online 的方式来访问对象属性，而实际上应该使用 allUsers[user].online 或 allUsers[user]['online']。

    • 第二段代码正确地使用 for...in 循环访问了对象的属性，并根据每个属性对应的对象中的 online 属性值进行计数。

      所以，第二段代码更为准确和可靠，能够正确计算符合条件的用户数量。
